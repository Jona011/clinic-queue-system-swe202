# 🏥 Clinic Queue and Appointment Management System (CQAMS)

## 🚀 About the Project
Developed as part of the **SWE202 – Software Modeling and Design** course, CQAMS is engineered to reduce patient waiting times, eliminate scheduling conflicts, and provide accurate operational visibility for clinical staff. The system operates on a client-server architecture, prioritizing security, high availability (99.5% uptime), and seamless user experience.

## ✨ Key Features
- **🔐 Role-Based Access Control (RBAC):** Secure, isolated environments tailored for different staff and user levels.
- **⏱️ Real-Time Queue Management:** Dynamic walk-in queue tracking with automated ticket generation and status updates.
- **📅 Smart Appointment Engine:** Self-service booking, modification, and intelligent conflict prevention.
- **📂 Medical Records Management:** Secure access to patient medical histories, clinical notes, and prescriptions.
- **🔔 Automated Notifications:** System-generated alerts for appointment confirmations, 24-hour reminders, and live queue status.
- **🌍 Bilingual Interface:** Native support for both English and Albanian languages.

## 👥 User Roles
The system is partitioned into four distinct operational roles:
1. **Administrator:** System governance, configuration, reporting, and audit log monitoring.
2. **Receptionist:** Patient registration, appointment coordination, and live queue management.
3. **Doctor:** Consultation management, clinical documentation, and schedule availability configuration.
4. **Patient:** Self-service portal for booking appointments and viewing personal medical records.
